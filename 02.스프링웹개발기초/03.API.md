# API

1. 정적 컨텐츠로 HTML로 바로 내리느냐
2. API방식을 사용해서 

- 뷰 이런거 없이 리턴의 문자가 그대로 내려감
```java

@GetMapping("hello-string")
@ResponseBody  
// 위에 ResponseBody 꼭 필요함
public String helloString(@RequestParam("name") String name) {
    return "hello " + name; // "hello spring"
}
// 뷰 이런거 없이 리턴의 문자가 그대로 내려감
```

- 만약 문자 말고 데이터를 주라고 한다면
(이 방식이 가장 많이 사용)
```java
    @GetMapping("hello-api")
    @ResponseBody
    public Hello helloApi(@RequestParam("name") String name) {
        Hello hello = new Hello();
        hello.setName(name);
        return hello;

    }
    static class Hello{
        private  String name;

        public String getName() {
            return name;
        }

        public void setName(String name) {
            this.name = name;
        }
    }
```

최근 프로젝트 할 때 json방식으로

- spring의 기본은 객체 반환 할 때 `@ResponseBody` 를 사용하고, 객체를 반환하면 객체가 JSON으로 변환됨

- `@ResponseBody` 를 사용
  - HTTP의 BODY에 문자 내용을 직접 반환
  - `viewResolver` 대신에 `HttpMessageConverter` 가 동작
  - 기본 문자처리: `StringHttpMessageConverter`
  - 기본 객체처리: `MappingJackson2HttpMessageConverter`
  - byte 처리 등등 기타 여러 HttpMessageConverter가 기본으로 등록되어 있음
  
- 참고: 클라이언트의 HTTP Accept 해더와 서버의 컨트롤러 반환 타입 정보 둘을 조합해서
`HttpMessageConverter` 가 선택된다. 더 자세한 내용은 스프링 MVC 강의에서 설명하겠다.
